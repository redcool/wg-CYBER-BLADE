## 一、武器系统

### 1.1 数值字段使用率

52 把武器，46 个字段，使用状态如下：

#### ✅ 完全使用（28字段）

id, name, desc, icon, cost, tag, minLevel, damage_lv1~4, cooldown_lv1~4, speedMult, critChanceAdd, critDamageAdd, armorAdd, hpRegenAdd, maxHpAdd, lifeStealAdd, bulletCount, bulletSpeed, bulletMaxRange, attackRange, spread, pierce, burnDps, burnMaxStacks, chainCount, splashRadius, homingStrength, slowAmount, slowDuration, healOnHit, auraHeal, auraRadius, sprayCone, behavior, class, knockback, magSize, reloadTime

#### ⚠️ 部分使用 / 死字段（4字段）

| 字段 | 问题 |

| `damageReductionAura` | 代码有完整实现，但 CSV 无此列 → 全武器 null |
| `killHeal` | 代码有实现，但 CSV 无此列 → 全武器 null |

### 1.2 行为分布

| 行为 | 数量 | 占比 | 说明 |
|------|------|------|------|
| bullet | 13 | 25% | 直射子弹，同质化严重 |
| melee_sweep | 8 | 15% | 近战挥砍，差异化尚可 |
| spread | 8 | 15% | 散射子弹，同质化严重 |
| melee_thrust | 6 | 12% | 近战突刺，尚可 |
| explode / frost / homing / spray | 各3 | 各6% | 元素类，差异化中等 |
| shield_aura | 2 | 4% | 光环，独特 |
| shock | 2 | 4% | 电击连锁，独特 |
| heal_bullet | 1 | 2% | 治疗弹，唯一 |

**bullet + spread = 21 把（40%）**，行为高度同质化。

### 1.3 协同断裂（严重）

- 武器 tag 词汇：`melee, gun, bow, magic, medic, lance`（6个）
- Synergy 词汇：`melee, ranged, fire, explosive, crit, tech, economy`（7个）
- **交集只有 melee** → 仅 ~8 把 melee 武器能激活协同
- `normalizeTag()` 是 identity 函数，没有映射实现

**后果**：gun/bow/magic/medic/lance 武器完全无法触发协同加成。

### 1.4 差异化问题

1. **40% 武器行为相同**（bullet + spread），仅靠数值区分
2. **弹匣机制只用在一把武器**（revolver），其余 51 把无限弹药
3. **缺乏真正独特行为**：无蓄力、反弹、分裂、弹射、地雷、召唤、双模式
4. **特性协同无运行时消费**：`burningSpread`, `chainExplosion`, `onCritLightning`, `goldToDamage` 有玩家属性但从不被代码读取

## 二、道具系统（严重问题）

### 2.1 CSV 列映射错误（致命）

CSV 列顺序与 schema 不匹配：

| 列 | CSV 注释意图 | 实际解析为 | 后果 |
|----|-------------|-----------|------|
| 第6列 | `qualifies`（条件表达式） | `unique: boolean` | 全 false，条件丢失 |
| 第7列 | itemClass（留空） | `rarity: string` | 全空 |
| 第8列 | rarity | `tags: array` | 全是 `["common"]` 等稀有度名 |
| 第9列 | tags（流派） | `triggers: array` | 存了触发器类型字符串 |
| 第10列 | effects | ✅ effects | 正常 |
| 第11列 | statMods | ✅ statMods | 正常 |

**后果**：道具的 `rarity`、`tags`（流派）、`unique`、`qualifies` 全部失效。

### 2.2 触发器全部不生效（致命）

| 道具 | CSV trigger | 运行时间 | 失败原因 |
|------|-----------|---------|---------|
| fury_blade | onKill | OnKill | camelCase ≠ PascalCase |
| energy_shield | onHit | OnHit | camelCase ≠ PascalCase |
| explosive_rounds | onHit | OnHit | camelCase ≠ PascalCase |
| replicator | onFire | 无此事件 | 无事件分发 |
| thorns_item | reflective | 无此事件 | 无事件 |
| tardigrade | revive | 无此事件 | 无事件 |
| ricochet_item | ricochet | 无此事件 | 无事件 |
| anvil | upgrade | 无此事件 | 无事件 |
| coupon_item | shopDiscount | — | 通过 statMods 勉强生效 |

**后果**：9 个有触发器效果的道具中，8 个完全无效。

### 2.3 道具标签断裂

- 无道具具有实际流派标签（melee/ranged/fire...）
- `countItemTags()` 对所有 7 个流派返回 0
- 流派检测偏向仅依赖武器，道具零贡献

### 2.4 差异化不足

- **81%（38/47）** 的道具是纯 statMod（数值加减）
- 仅 19%（9/47）带触发器，且全不生效
- 多个道具影响同一属性但无独特机制
